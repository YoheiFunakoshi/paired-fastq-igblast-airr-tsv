# RG社Excelとの比較注釈（KKF103hG）

作成日: 2026-05-03

この文書は、`RG Paired Fastq IgBLAST AIRR tsv` の出力をRG社納品Excelと比較した際の注釈です。GitHubにはFASTQ、RG社作成Excel、AIRR TSVなどの実データは置かず、比較結果の要点と判断だけを記録します。実データ一式は船越が保有しています。

## 最終判断

この版では、ユニークリードを次の3要素で定義します。

- アリルを外したVコール候補セット
- アリルを外したJコール候補セット
- `junctionAA` / `final_junction_aa`

D callは不安定でコールされないことが多いため、ユニーク判定には使いません。C領域（IGHG1、IGHG2、IGHG3など）も、この暫定版ではユニーク判定に使いません。C領域で分けるとRG社Excelにより近づく可能性はありますが、今回は「落とさないこと」「シンプルで追跡可能であること」を優先し、これ以上複雑化しない方針としました。

## RG社Excelとの主要比較

KKF103hGで確認した主な数値は次の通りです。

| 項目 | RG社Excel | 本ツール |
|---|---:|---:|
| Total reads / read pairs | 280,262 | 280,262 |
| In frame / counts対象 | 99,974 | 107,568 |
| Unique reads / final clonotypes | 26,454 | 26,017 |

総リード数は一致しました。ユニーク数はRG社 26,454 に対して本ツール 26,017 で、差は437、約1.65%です。この差は実用上大きな乖離ではなく、V/J/junctionAAを中心に解析する目的では許容範囲と判断しました。

## 上位クローンの照合

RG社Excelの`Back_data`シートにはサマリーとして26,454ユニークが記載されていますが、実際に行として確認できる`All Data`表は上位212行でした。そのため、全ユニークの完全な1対1照合ではなく、上位クローンの照合として確認しました。

上位212行に対する対応は次の通りです。

| 判定 | 一致数 |
|---|---:|
| CDR3AA一致 | 210 / 212 |
| J + CDR3AA一致 | 204 / 212 |
| J + CDR3AA一致、かつV候補に重なりあり | 200 / 212 |
| V候補セット + J候補セット + CDR3AAが完全一致 | 147 / 212 |

V候補セットまで完全一致にすると一致数は下がります。主な理由は、RG社が`IGHV2-70D,IGHV2-70`のように複数候補を表示する一方、本ツールではR2優先の暫定final V callにより`IGHV2-70`のようにまとまる例があるためです。J+CDR3AA、CDR3AAのレベルではよく対応していました。

## RG社Excelとの主な違い

RG社ExcelにはIGHC、つまりIGHG1、IGHG2、IGHG3などのC領域情報があります。同じV/J/CDR3AAでもC領域別に行が分かれる例がありました。一方、本ツールの`integrated_counts.tsv`では、C領域はユニーク判定に使いません。

この違いにより、RG社Excelでは複数行、本ツールでは1行にまとまるクローンがあります。これは仕様上の差であり、現時点では致命的な問題とは判断していません。将来、C領域別解析を主目的にする場合は、`final_c_call`またはC領域推定列を追加し、C領域を集計キーに含める版を別途作る余地があります。

## C領域を使わないのにユニーク数が近い理由

C領域をユニーク判定に使わない場合、ユニーク数が大きく減るのは、同じVコール、同じJコール、同じ`junctionAA` / `final_junction_aa`を持つクローンが、IGHG1、IGHG2、IGHG3など複数のC領域にまたがっている場合だけです。

例えば、RG社Excelで次のようにC領域別に3行に分かれていても、本ツールでは1行にまとまります。

| V | J | junctionAA | C |
|---|---|---|---|
| IGHV1-69 | IGHJ2 | CARE...FDLW | IGHG1 |
| IGHV1-69 | IGHJ2 | CARE...FDLW | IGHG2 |
| IGHV1-69 | IGHJ2 | CARE...FDLW | IGHG3 |

一方で、C領域が異なるクローンでも、V/J/junctionAAのいずれかが異なれば、本ツールでも別ユニークとして数えられます。そのため、C領域を使わないことによるユニーク数の減少は、全クローンに一律に起こるわけではありません。

KKF103hGでは、RG社Excelのユニーク数26,454に対して本ツールのユニーク数は26,017で、差は437、約1.65%でした。このことから、少なくともこのデータでは、C領域をユニーク判定に入れないことによる影響は限定的であり、V/J/junctionAAベースの解析目的には十分近い結果と判断しました。

## この版で完成とする理由

このプロジェクトの目的は、R1/R2をマージせずにAIRR TSVを作成し、R1由来・R2由来の生AIRRデータを残したまま、共同研究者が扱いやすい暫定統合TSVと集計表を得ることです。

現時点で、総リード数は一致し、Vコール、Jコール、junctionAAで定義したユニークリード数もRG社Excelと概ね近い結果になりました。そのため、この版をいったん完成版とします。

より厳密な正確性を追う版、例えばR1/R2 conflictを厳密に扱う版、C領域で分ける版、SHMやcomplete VDJを重視する版は、今後の別バージョンとして検討します。
